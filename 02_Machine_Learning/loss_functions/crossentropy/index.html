Mathematical Foundation

CrossEntropy loss measures the difference between two probability distributions. For classification problems, it measures how well our model's predicted probabilities match the true distribution.

Binary CrossEntropy:

$$H(p,q) = -[p \cdot \log(q) + (1-p) \cdot \log(1-q)]$$
Multi-class CrossEntropy:

$$H(p,q) = -\sum_{i=1}^{C} p_i \cdot \log(q_i)$$

Where:

  • $p$ is the true probability distribution
  • $q$ is the predicted probability distribution
  • $C$ is the number of classes
  • $p_i$ and $q_i$ are the probabilities for class $i$

Why CrossEntropy for Classification?

  • Probability Interpretation: Works directly with probability distributions
  • Gradient Properties: Provides strong gradients when predictions are wrong
  • Convexity: Guarantees global minimum for logistic regression
  • Information Theory: Based on entropy and information theory principles

Derivation from Information Theory

CrossEntropy comes from information theory. The entropy of a probability distribution is:

$$H(p) = -\sum_{i=1}^{C} p_i \log(p_i)$$

CrossEntropy measures the additional information needed to encode data using the wrong distribution:

$$H(p,q) = -\sum_{i=1}^{C} p_i \log(q_i)$$

The gradient of CrossEntropy with respect to the logits $z_i$ is:

$$\frac{\partial H}{\partial z_i} = q_i - p_i$$

This elegant gradient form makes CrossEntropy ideal for neural network training!

Quick Example

Here's a simple example of how CrossEntropy loss works:

Example:

True label: $[0, 1, 0]$ (class 1)

Model prediction: $[0.1, 0.8, 0.1]$

CrossEntropy loss: $-\log(0.8) \approx 0.223$

When the model is confident and correct, the loss is low. When the model is wrong or uncertain, the loss increases exponentially.
